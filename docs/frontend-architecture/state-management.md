# 状態管理

## クライアント状態

- React標準hooks（`useState`, `useReducer`）を使用
- Next.jsに依存しないこと
- `use client`や`next/~~~~`を用いた実装を行わない

---

## サーバー状態

- page.tsx（SSR）で初期データ取得を行う

---

## グローバル状態

### 方針

- 現時点ではJotai等の状態管理ライブラリは導入しない
- `packages/ui`（汎用コンポーネント）には状態管理ライブラリを入れない。props in / callback outを維持する
- `apps/`側も現状は`useState` + propsで十分制御できているため不要

### 導入を検討するタイミング

- FSD等でアプリケーションレイヤーごとにUIを持つ構成に変わり、離れたレイヤー間での状態共有が必要になった時
- props drillingが4-5階層以上に深くなった時
- 認証/ユーザー情報など、アプリ全体で参照する状態が増えた時

### 理由

- atomベースのライブラリ（Jotai等）はグローバルスコープになるため、依存関係が暗黙的になりデータフローが追いにくくなる
- 現状のアーキテクチャ（`packages/ui` + `apps/`のページ単位構成）ではprops明示の方が見通しが良い

---

## 既存実装の課題

- インタラクティブなコンポーネントは`use client`を用いて実装
  - CSRのインターフェースを介する検討が必要
- `next/Link`、`next/Image`などが使用できないため最適化されていない
  - Reactベースでの代替実装が必要
